Binutils

The Binutils (or the GNU Binary Utilities) is a package of programming tools for creating and managing binary programs, object files, libraries, profile data, and assembly source code.

Most of these programs use BFD - the Binary File Descriptor library (libbfd) which provides support for various object file formats. Many of them also use the opcodes library to assemble and disassemble machine instructions.

The main tools in this pkg are:

as

the GNU Assembler; in fact, it's a family of assemblers; even if you never run it explicitly, it's invoked implicitly when you run gcc, because the main purpose of as is to assemble the output of the GNU C compiler for use by ld;

ld

runs the GNU linker which creates executable files (or libraries) from object files created during compilation; ~ combines a number of objects files, relocates their data and ties up symbol references; usually it's invoked implicitly when you run gcc or make;

There are also dynamic linker/loader ld-linux.so.2 and its older incarnation ld.so (the first handles ELF, the last handles outdated a.out files). Both are not related to binutils!

Note that ld is used during software build process only, while ld-linux.so.2 is required each time when a program starts - the dynamic linker/loader finds and loads shared libraries required by the program. Of course it's not necessary if all required libs were statically linked during app build.

The binutils package also includes:

addr2line

converts addresses into filenames and line numbers;

ar

creates, modifies, and extracts from archives;

c++filt

filter to demangle encoded C++ symbols;

dlltool

creates files for building and using DLLs (Windows dynamic-link libraries);

gold

alternative linker (a new, faster, ELF only linker) (still beta?);

gprof

profiler (displays profiling information);

nlmconv

converts object code into an NLM (NetWare Loadable Module); I thought NetWare is long forgotten;

nm

lists symbols from object files;

objcopy

copies and translates object files;

objdump

displays information from object files;

ranlib

generates an index to the contents of an archive;

readelf

displays information from the ELF-format object files;

size

lists the section sizes of an object or archive file;

strings

lists printable strings from files;

strip

removes symbols from an object file;

windmc

A Windows compatible message compiler (generates Windows message resources);

windres

a compiler for Windows resource files;

Usage examples:

readelf -d /lib/x86_64-linux-gnu/libz.so.1

display contents of the dynamic section of the specified file (libz.so.1);

readelf -l /lib/x86_64-linux-gnu/libz.so.1

display segment's headers of the specified file;

size /usr/bin/mplayer

print section sizes of the specified file;

strings --data /usr/bin/mplayer | more

output the strings [of printable chars] from the initialized, loaded data sections in the specified file; by default only strings of 4+ chars are selected;
