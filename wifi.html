Abbreviations

AP Access Point. AP is a wireless network device that connects laptops, smartphones, WiFi adapters, etc to a local area network.
BSS Basic Service Set. There is one Access Point for each BSS and they talk to each other only if they belong to the same ESSID.
BSSID Basic Service Set ID. This is a 48-bit number used to identify the BSS.
DSSS Direct Sequence Spread Spectrum. This a physical layer protocol (Open Systems Interconnect, layer 1)
ESS Extended Service Set. This is a specific WLAN with a specific SSID, and it may consists of several BSS.
ESSID Extended Service Set ID. ESSID helps to prevent the chaos. Each wireless network needs to be assigned an ESSID that doesn't match that of any other network within its range of operation. The desired ESSID is then set on all wireless NICs and APs belonging to this particular WLAN, and they ignore all traffic using other identifiers.
FHSS Frequency Hopping Spread Spectrum. This a physical layer protocol (Open Systems Interconnect, layer 1)
PSK Pre Shared Key mode
SSID Service Set ID
TKIP Temporal Key Integrity Protocol
WEP Wired Equivalent Privacy. It was the first widely used data encryption scheme for wireless networks. A flaw in the encryption scheme was soon discovered and freely available tools like "WEP crack" became available to decipher WEP encryption keys within minutes.
WPA WiFi Protected Access. This is a newer better security protocol that was introduced to replace WEP.

Access Points and Routers

A home net usually needs a WiFi router.

Access Points are mostly for companies and larger networks.

WiFi router acts as a hub that sets up a local area network and provides wired and wireless connections for the devices. Access Point (AP) is a wireless network device that connects laptops, smartphones, WiFi adapters, etc to a local area network. You can also consider Access Point as a sub-device within the local area network that can be used to extend the wireless coverage of an existing network and/or to allow the larger number of users to connect to that network.

The term 'Access Point' is widely used in wireless technology, and a hardware device providing the functionality of 'Access Point' (as described above) is just a case.

Note that WiFi router is an Access Point by definition. But opposite is not true: Access Point (hardware device) cannot function as a router. Routers manage local area networks, provide communications with other networks, ensure security, etc. Access Points typically only provide access to the networks established by routers.

In general, routers and routing is a very large and complicated area of knowledge. But WiFi routers reside on the simpler and cheaper end of the router family. A WiFi router can be roughly defined as a network device providing two main services:

  • It connects multiple devices (computers, phones, tablets, printers, etc) to a Local Area Network (which is often a home network).
  • It provides Internet access for all those connected devices.

As of the year 2021, a typical WiFi router provides 1000 Mbit/s Ethernet ports for wired connections and dual-band wireless access, supports 802.11ac (WiFi 5), provides many other functions like DHCP, Firewall, etc.

Antennas direction and router location

WiFi coverage is limited, walls and other obstacles decrease the signal. So, it is important to find an optimal location for your WiFi router. The universal rule: it should be equidistant from all the signal sources, whatever it means in your case. Probably, somewhere in the center of the house or office space.

Also, you should choose the optimal orientation for your router's antennas. By default, antennas must be pointing straight to the sky. To be more exact: position antennas upward for a better horizontal reach; position antennas sideways for better vertical reach (e.g. in a two-story house).

About walls and other obstacles

The WiFi signal penetrates the obstacles and tries to go around the obstacles. The more obstacles, the worse for the signal. Obstacles may cause absorbtions and reflections of the signal, and sometimes the result is unpredictable.

The open space is the best case.

Some materials (e.g. wood, plastic, ordinary glass) are quite penetrable for the wireless signals. Others, like tinted glass, water (large volumes), brick, plaster are materials with the medium absorbtion. The worst obstacles are concrete (walls, floors, ceilings), metal (e.g. iron doors, metal nets, grates), ceramics.

Here is a table showing how much some obstacles decrease the WiFi range (2.4 GHz band). Note that these numbers are approximate, just for the better understanding of the problem.

Material The range in percents of the theoretically possible
Open space (air) 100%
Window / glass 70%
Window / tinted glass (has metal constituents) 50%
Wooden wall 30%
Concrete interior wall ~15cm/6ft 15%
Concrete bearing wall ~30cm/12ft 10%
Concrete floor/ceiling 10-15%

Some other recommendations:

  • Keep your router away from the massive metal objects - they effectively dissipates electromagnetic energy. This also applies to the large volumes of water (e.g. big aquarium).
  • The potential sources of interference in the 2.4 GHz band:
    • Microwave ovens
    • 2.4 GHz cordless phones, DSSS and FHSS
    • Fluorescent bulbs
    • 2.4 GHz video cameras
    • Elevator motors
    • Cauterizing devices
    • Plasma cutters
    • Bluetooth radios
    • Nearby 802.11, 802.11b or 802.11g WLANs
    • Wireless Internet Service Providers (WISPs)
  • The potential sources of interference in the 5 GHz band:
    • 5 GHz cordless phones
    • Radars
    • Perimeter sensors
    • Digital satellites
    • Nearby 802.11a or 802.11n WLANs
    • Outdoor wireless 5 GHz bridges
  • Some people are afraid of WiFi routers (radiates electromagnetic waves). WiFi routers don't like people too - humans absorb electromagnetic waves. So, try to keep the reasonable distance between those entities.
  • If possible, mount your WiFi router high, i.e. over the obstacles.

Bands

WiFi frequency bands: 2.4 GHz and 5 GHz.

Frequency bands are ranges of radio wave frequencies used to transmit data in the wireless spectrum. These bands can further be broken down into WiFi channels. The general rule: higher frequency provides faster data transmission, but shorter signal range.

The main differences between 2.4 GHz and 5 GHz bands are:

  • Network coverage. 2.4 GHz band provides better coverage than 5 GHz. The lower frequencies can more easily penetrate solid objects, like walls.
  • Network speed. 5 GHz band is faster than 2.4 GHz, it provides up to 1300 Mbps, while 2.4 GHz band support speeds between 450 Mbps and 600 Mbps at best.
  • Co-channel interference. In the 2.4 GHz band, you can choose from 11 WiFi channels, and only 3 of them are non-overlapping. In the 5 GHz band, you can choose from 45 channels, where 24 are non-overlapping. Overlapping channels suffer from interference. Also, 2.4 GHz band is used by some household appliances, and this can make the situation even worse.

Channels

WiFi channels are smaller bands within WiFi frequency bands that are used to send and receive data. In general, you can have:

  • 11 WiFi channels in the 2.4 GHz frequency band
  • 45 WiFi channels in the 5 GHz frequency band

Some WiFi channels are better than others because of

  • co-channel interference - devices are competing for time to talk on the same channel;
  • adjacent-channel interference - devices from overlapping channels are trying to talk over each other;

You can avoid such interference by choosing to use a non-overlapping, uncrowded WiFi channel. Of course, if there is a possibility to choose from something.

Each WiFi channel has a 20 MHz segment in the frequency band. Several channels can be bonded together to form wider segments. The width of a WiFi channel dictates the throughput. Or, simply speaking, the wider channel - the faster speed, and more data transferred. Unless they are impacted by interference.

Choosing a WiFi channel width

Interference is more common in certain frequency bands on certain channels. There are some recommended channel width settings for routers and access points:

  • Set 2.4 GHz WiFi channel width to 20 MHz. Because there are several overlapping channels in this band - 8 out of 11. This can cause an interference, and, more likely than not, a weaker wireless performance instead of the expected faster speeds.
  • Set 5 GHz WiFi channel width to 20, 40, or 80 MHz. This band has significantly more channels, and 24 out of 45 do not overlap. So, it is less crowded and better equipped to support wide channels.

When adjusting channel width settings for 5 GHz band, you should enable support for all channel widths. Otherwise devices supporting 20 MHz channels only, won't be able to connect to a router with 80 MHz-only mode.

SSID, BSSID, ESSID

SSID - for regular users, BSSID and ESSID - for pros.

The terms BSSID, ESSID, and SSID are used to describe sections of a wireless network (WLAN). They have slightly different meanings.

A regular user needs to know SSID to connect to a wireless network. SSIDs are names of wireless networks.

Administrators and pros should probably know about BSSIDs and ESSIDs.

SSID

Service Set is just another term referencing a wireless network (WLAN).

Since WLANs can coexist in the same area/airspace, each WLAN needs a unique name. This name is SSID (the Service Set ID of the network). SSIDs serve as WiFi network names and are typically natural language labels. The SSID is broadcast by stations in beacon packets to announce the presence of a network. A mobile device looks for all networks in the range when you try to connect to a local WiFi.

To connect to a specific WLAN, you need to know its SSID and password.

BSSID

BSS (Basic Service Set) is the area covered by one Access Point.

The Access Points within WLAN are identified by BSSID - a Basic Service Set Identifier. BSSID is included in all wireless packets. By convention a MAC address of an Access Point is used as BSSID, i.e, if you know the MAC address, you know the BSSID, and, since all packets contain the originator's BSSID, you can usually trace those packets. This works fine for an access point with one radio and one WLAN configured.

Most often, there are different BSSIDs on an Access Point for each WLAN configured on a radio. If you have an AP with 2 radios and 32 WLANs configured on each, you would have 64 BSSIDs plus the base access point BSSID. To accommodate the multiple BSSIDs, each AP is assigned a unique block of 64 MAC addresses. Each radio has 32 MAC addresses and supports up to 32 service set identifiers (SSIDs), with one MAC address assigned to each SSID as a basic service set identification (BSSID). All MAC addresses for an access point are assigned based on the base MAC address of the AP (usually the Access Point MAC address block is listed on a label on the back of the AP).

ESSID

ESS (Extended Service Set or Extended Basic Service Set) consists of all BSSs in the network. Technically, ESSID identifies the same network as the SSID does. The term SSID is used most often.
